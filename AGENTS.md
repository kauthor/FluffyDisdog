# FluffyDisdog — AGENTS.md

## 프로젝트 개요

**게임명:** FluffyDisdog (땅부수기)
**장르:** 타일 파괴 + 덱빌딩 로그라이크 (엔드리스 지향)
**개발 단계:** 알파 내부 테스트
**엔진:** Unity (C#)

### 핵심 게임 구조
- 8x8 타일 보드에서 카드를 사용해 타일을 파괴하여 목표 점수 달성
- 스테이지 클리어마다 카드 획득/제거/강화로 덱 최적화
- 현재 플레이어블 캐릭터: 모험가 1종 (추가 캐릭터 확장 예정)
- 크로스 보상: 스테이지 보상에서 타 캐릭터 카드/유물 획득 가능

## 기획 규칙 — 에이전트 판단 기준

### 카드 시스템
- 도구 태그 카드: 사용 시 파괴 확률 존재 (기본 0%, 사용마다 +5%, 최대 70%)
- 강화(+) 카드: 파괴 확률 제거, 효과 상향
- 카드 태그는 비트 플래그 방식 (여러 태그 동시 보유 가능)
- 소각 태그(128비트): 사용 즉시 덱에서 영구 제거
- 핸드 기본 8장 (유물로 변동 가능)

### 타일 상태 시스템
- **균열**: 타격 시 파괴 확률 무시, 반드시 파괴 / 자연 소멸 없음 / 방해 타일 부여 불가
- **전염**: 매 턴 근접 1칸 전파 (미확정 항목 있음)
- **암흑**: 태그/파괴 여부 확인 불가 / 횃불 카드로 해제
- **잔상**: 파괴된 타일 위치에 생성 / 다음 턴 소멸 / 기본 점수 0점

### 유물 시스템
- 수치는 RelicData.Values[] 배열로 관리 (엑셀에서 수정 가능)
- 현재 구현된 유물: relic101~relic122 (22종)
- 새 유물 추가 시: RelicName 열거형 → CS 파일 → 팩토리 등록 → 테이블 추가

### 의뢰 시스템 (현재 빌드)
- 이번 빌드: 항상 성공 처리 (100%)
- 추후 구현: 실패/대성공 분기, 성공률 누적, 투자 시스템
- 등급: Normal(5G) / VIP(15G) / VVIP(30G)

### 상점 시스템
- 상점 가격: costMin~costMax 사이 랜덤 결정
- 리롤: 낱장 카드 제외 전체 가능
- 의뢰 슬롯: 항상 1개 고정
