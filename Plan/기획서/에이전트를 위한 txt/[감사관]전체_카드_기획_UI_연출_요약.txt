[기획 일관성 감사관용] 전체 카드 기획 및 UI 연출 요약
출처: 카드_정리.pptx, 땅부수기2026_ch2.pptx
========================================

■ 모험가 캐릭터 기획 의도
- 빠른 덱 압축 + 원하는 카드 강화 + 골드 수급이 핵심
- 도구 태그 카드를 주로 사용하도록 설계
- 시작 카드 15장: 삽 8장, 갈퀴 2장, 도끼 2장, 황금 삽 1장, 가방 1장, 도구 강화 1장

■ 카드 태그 정의
- #도구: 사용 시 카드 파괴 확률 존재 (덱 압축 메커니즘)
- #황금: 카드 사용 시 골드 추가 획득
- #소모형: 타일 타격 없이 특수 효과만 발동
- #강화: 강화된 카드 버전, 도구 파괴 효과 제거

■ 전체 카드 목록 및 기획 의도

[도구 태그 카드]
- 삽: 1칸 100% 파괴 / 기본 모험가 카드
- 삽+: 1칸 100% + 방해 블록 파괴
- 갈퀴: 중심 100% + 주변 66% / 다중 영역 파괴
- 갈퀴+: 전 범위 100%
- 도끼: 중심 100% + 주변 66% / 갈퀴와 타격 범위 상이
- 도끼+: 전 범위 100%
- 황금 삽: 1칸 100% + 골드 획득 / 모험가 전용 황금 카드
- 황금 삽+: 파괴 효과 제거 + 골드 획득 유지
- 황금 도끼: 중심 100% + 주변 66% + 골드 획득
- 황금 도끼+: 전 범위 100% + 골드 획득
- 나무 망치: 중심 100% + 주변 50% / 범위 타격 도구
- 나무 망치+: 중심 100% + 주변 75%
- 곡괭이: 중심 100% + 주변 50% / 나무망치와 타격 범위 상이
- 곡괭이+: 중심 100% + 주변 75%
- 새총: 랜덤 5칸 33% 파괴 / 불확실성 기반 카드
- 새총+: 랜덤 7칸 50% 파괴
- 모서리 파쇄 해머: 중심 100% + 상하좌우 66% + 대각선 33%
- 모서리 파쇄 해머+: 중심 100% + 상하좌우 75% + 대각선 50%
- 집중형 십자드릴: 중심 100% + 대각선 66% + 상하좌우 33%
- 집중형 십자드릴+: 중심 100% + 대각선 75% + 상하좌우 50%
- 대형 가위: 중심 100% + 주변 75%
- 대형 가위+: 전 범위 100%

[특수 도구 카드]
- 모험가의 횃불: 암흑 타일 밝힘 + 도움/방해 타일 골드 획득 + 다음 황금 카드 효과 2배
- 모험가의 횃불+: 위 효과 + 암흑 방해 타일 파괴
- 기억의 망치: 잔상 타일 타격 시 인접 2칸 100% 파괴
- 기억의 망치+: 잔상 타일 타격 시 인접(대각 포함) 4칸 100% 파괴
- 균열 지층 분석: 3칸 이상 파괴 시 전체 파괴, 균열 타일당 2G, 5칸 파괴 시 황금 카드 드로우
- 균열 지층 분석+: 균열 타일당 5G, 5칸 파괴 시 황금 카드 2장 드로우

[소모형 카드]
- 가방: 덱 최상단 1장 드로우
- 도구 변형: 손패 도구 카드를 덱 내 도구 카드로 교체
- 도구 강화: 다음 턴 도구 카드 점수 2배

[약화 카드 - 시작 카드]
- 약화 카드(일반): 범위 내 일반 타일 균열 상태로 변경
- 약화 카드(레어): 동일 효과, 더 넓은 범위

■ 균열 타일 상태 규칙
- 균열 타일: 다음 턴 타격 시 파괴 확률 무시하고 반드시 파괴

■ 잔상 타일 상태 규칙
- 파괴된 타일 위치에 생성
- 타격 지점으로 사용 가능
- 다음 턴에 사라짐
- 기본 점수 0점

■ 암흑 타일 상태 규칙
- 해당 타일 태그 및 파괴 여부 확인 불가
- 횃불 카드로 해제 가능

■ UI 연출 상세 (땅부수기2026_ch2.pptx)
- 타일 타격 연출: 좌우 1px 흔들림 + 파편 파티클
- 파괴 실패 연출: 상하 1px 흔들림 + 작은 먼지 파티클
- 점수 정보표: 태그 점수(좌) / 타일 점수(우) 분리
- 덱/묘지 보기: DECK / CARDS / TRASH 구분 표기
- 불안정 시스템: 방해 타일 타격 시 게이지 +1, 10 달성 시 게임 오버
  - 불안정 룬: 최솟값 상향으로 난이도 증가
  - 무너지는 발판: 불안정 초기화 불가

■ 의뢰 시스템 상세 (기획)
- 상점 입장 시 의뢰 가능 (슬롯 1개 고정)
- 등급: 일반(5G) / 희귀(15G) / 유니크(30G)
- 투자: 동일 금액 추가 지불 → 성공/대성공 확률 상승
- 보상(실패): 등급 카드 1장 / 랜덤 유물 / 랜덤 소모품 / 카드 제거·강화 중 1택
- 보상(성공): 랜덤 카드팩 / 태그 관련 유물 / 카드 제거·강화 중 1택
- 보상(대성공): 태그 관련 카드팩 + 태그 관련 유물 + 카드 제거·강화 전부 지급

■ 의뢰 시스템 구현 명세 (작성일: 2026-05-10 / 빌드: 부분 구현)

[이번 빌드 구현 범위]
- 의뢰 성공 처리: 항상 성공 (100%) 처리
- 플랜별 비용 차별화: Normal 5G / VIP 15G / VVIP 30G
- 플랜별 보상 차별화: Success_box_id 참조
- 보상 화면: UICardPackResultPopup 재사용

[추후 구현 예정]
- 실패 분기 및 화면 (Fail_box_id 참조)
- 대성공 분기 및 화면
- 방문당 성공률 누적 로직
- 투자 시 성공률 증가 로직
- 대성공 가중치 보상 (RequestWeightData 활용)
- TradersCompanion 유물 연동

[구현 흐름]
1. 의뢰 시작: 플레이어가 등급 선택 → RequestData.Cost 참조 → 골드 차감
2. 보상 수령: RequestData.Success_box_id → BoxData.Pick_Count 확인 (0 = 전부 지급)
             → BoxItemData 아이템 순회 → Reward_type 6(가챠) → GachaData 랜덤 추출
             → UICardPackResultPopup 표시

[테이블 구현 현황]
- 06_PackData: 완료
- 07_GachaData: 완료 (의뢰 보상용 Gacha_Id 10101/10201/10301 추가 필요)
- 08_ShopItemData: 완료
- 09_ShopData: 완료
- 10_MapData: 진행 중 (2번 맵 이름 미확정 스테이지)
- 11_RequestData: 구현 필요 (RequestTable.cs 작성, ExcelManager 로드 추가)
- 12_RequestWeightData: 이번 빌드 스킵 (대성공 미구현, 데이터만 준비)
- 13_BoxData: 구현 필요 (테이블 로드 및 참조 로직)
- 14_BoxItemData: 구현 필요 (보상 지급 로직, Reward_type 6 연결)

[수정 대상 파일 - 우선순위 높음]

- UIStoreView.cs: 의뢰 비용 deg*5 → RequestData.Cost 참조로 수정
