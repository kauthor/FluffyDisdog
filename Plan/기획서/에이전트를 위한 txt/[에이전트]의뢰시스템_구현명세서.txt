# [참고 문서] 의뢰 시스템 구현 명세서 (Agent-Ready)

**파일명:** 의뢰시스템_구현명세서.docx  
**시스템 개요:** 플레이어가 골드를 지불하고 특정 등급의 의뢰를 수행하여 보상 상자(아이템/카드/가챠)를 획득하는 시스템

---

## 1. 구현 상태 및 범위 정의

| 항목 | 상태 | 내용 |
| :--- | :--- | :--- |
| **의뢰 등급** | **확정** | Normal, VIP, VVIP 3단계 구성 |
| **성공 로직** | **확정 (임시)** | 현재 빌드에서는 성공률 100% 고정 (Success_rate = 10000) |
| **비용 지불** | **확정** | `RequestData`의 `Cost` 컬럼 참조하여 골드 차감 |
| **보상 프로세스** | **확정** | 상자(Box) ID 참조 → 아이템 목록 순회 → 보상 지급 |
| **가챠 연동** | **확정** | Reward_type 6인 경우 `GachaData` 연동 및 전용 UI 팝업 출력 |
| **실패/대성공** | **보류** | 실패 분기 및 가중치 기반 대성공 로직은 차후 구현 |
| **외부 시스템 연동** | **검토 중** | 유물(TradersCompanion)에 의한 확률 변동 로직 |

---

## 2. 시스템 논리 흐름 (에이전트 판단 기준)

AI 에이전트가 의뢰 프로세스를 처리할 때 다음 순서를 따릅니다.

1.  **의뢰 시작 판단:**
    *   플레이어가 선택한 등급(`Req_grade`) 확인.
    *   `RequestData` 테이블에서 해당 등급의 `Cost` 값 확인.
    *   **조건:** 플레이어 소유 골드 ≥ `Cost` 일 때만 진행. 미달 시 에러 처리.
2.  **성공 여부 결정:**
    *   현재 빌드: 무조건 성공 처리.
    *   (차후) `Success_rate`와 난수 비교 로직 추가 예정.
3.  **보상 식별:**
    *   성공 시 `Success_box_id`를 참조하여 `BoxData` 및 `BoxItemData` 조회.
4.  **아이템 지급 처리:**
    *   `BoxItemData`의 `Reward_type`에 따라 분기 처리 (3번 항목 참조).
    *   `Reward_type 6`인 경우 `GachaData`에서 아이템을 추출하고 `UICardPackResultPopup`을 호출함.

---

## 3. 데이터 테이블 정의 및 참조 관계

### 3.1 주요 테이블 명세

| 테이블 번호 | 테이블명 | 주요 역할 | 에이전트 참조 컬럼 |
| :--- | :--- | :--- | :--- |
| **06** | **RequestData** | 의뢰 기본 정보 및 비용 | `Req_grade`, `Cost`, `Success_box_id` |
| **07** | **GachaData** | 가챠 아이템 풀 구성 | `Gacha_id`, `Item_id`, `Probability` |
| **13** | **BoxData** | 보상 상자 마스터 | `Box_id`, `Pick_count` (0은 전체 지급) |
| **14** | **BoxItemData** | 상자 내 상세 아이템 | `Box_id`, `Reward_type`, `Reward_value` |

### 3.2 Reward_Type 인덱스 (Enum 관리 권장)

에이전트가 보상을 생성할 때 사용하는 타입 인덱스입니다.

*   **1:** 재화 (Currency)
*   **2:** 카드 (Card)
*   **3:** 유물 (Relic)
*   **4:** 경험치 (EXP)
*   **5:** (예약됨 - 사용 금지)
*   **6:** 가챠 (Gacha) - `GachaData` 참조 필수

---

## 4. 시스템 간 연관 관계

*   **재화 시스템 (Currency System):** 의뢰 시작 시 골드 차감 연동.
*   **인벤토리/카드 시스템:** 보상 획득 시 해당 데이터 테이블에 아이템 추가.
*   **UI 시스템:** 
    *   의뢰 선택 화면.
    *   `UICardPackResultPopup`: Reward_type 6 처리 시 재사용.
*   **유물 시스템 (Relic System):** 차후 `TradersCompanion` 유물 보유 여부에 따라 성공률 보정 로직 연결 예정 (현재는 미연결).

---

## 5. 개발 시 주의사항 (에이전트 가이드)

*   **성공률 고정:** 현재 `Success_rate` 값에 관계없이 코드는 항상 성공으로 작동해야 함을 유의하십시오.
*   **예약 인덱스:** `Reward_type` 5번은 데이터 입력 방지를 위해 비워둔 값이므로, 로직 작성 시 이를 건너뛰거나 예외 처리를 확인하십시오.
*   **상자 지급 방식:** `BoxData`의 `Pick_count`가 `0`일 경우, `BoxItemData`에 등록된 해당 상자의 모든 리스트를 지급합니다.
