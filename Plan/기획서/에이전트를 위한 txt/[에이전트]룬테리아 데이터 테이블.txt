# [에이전트용 참조 문서] 룬테리아 데이터 시스템

## 1. 게임 기본 개요
*   **장르:** 타일 파괴 + 덱빌딩 로그라이크 (엔드리스 지향)
*   **핵심 루프:** 8x8 타일 보드에서 카드를 사용하여 타일을 파괴하고, 목표 점수를 달성하여 스테이지를 클리어함.
*   **상태:** [확정] 모험가 캐릭터 1종 플레이 가능 / [검토 중] 마법사 및 추가 캐릭터 확장

## 2. 카드 시스템 (02.CardData)
카드는 플레이어가 타일을 파괴하거나 특수 효과를 발동하는 핵심 수단입니다.

### 2.1 데이터 테이블 컬럼 정의
| 컬럼명 | 의미 | 주요 값 설명 |
| :--- | :--- | :--- |
| **ID** | 관리용 고유 아이디 | 100번대: 모험가/공용 카드, 130번대 이후: 더미/테스트 |
| **Card_type** | 카드의 근본 분류 | 1: 타격용 카드, 2: 소모품 카드 |
| **Char_type** | 캐릭터 전용 분류 | 1: 공용, 2: 모험가, 3: 마법사 |
| **Card_Upgraded_Type** | 강화 상태 | 0: 강화 불가, 1: 일반(강화 가능), 2: 강화 완료(+) |
| **Card_Upgraded_Key** | 강화/이전 카드 연결 | 강화 전후의 Card_Key를 서로 참조하여 데이터 연결 |
| **Card_Rarity** | 희귀도 | 1: 노말, 2: 레어, 3: 유니크 |
| **Char_bit** | 캐릭터 사용 권한 | 1: 모험가, 2: 마법사, 3: 공용 |
| **Tag_bit** | 카드 속성 태그 (합산) | 32: 도구, 64: 황금, 128: 일회성, 256: 마나, 512: 마법 |
| **Mana** | 사용 마나 (마법사) | 정수 값 (현재 모험가는 0 사용) |
| **Card_Add_Id** | 추가 데이터 참조 | `04.CardAddData` 테이블의 상세 효과 아이디와 연결 |

### 2.2 카드 상태 구분
*   **[확정]** 삽, 갈퀴, 도끼, 곡괭이 등 '도구' 태그 중심의 기본 카드군.
*   **[확정]** 강화 시스템: 일반 카드(예: 삽)는 강화 카드(예: 삽+)와 1:1 매칭됨.
*   **[보류]** 그레이트 소드, 실버 스피어 등 Dummy(ID 135~142) 카드는 현재 테스트용 데이터로 실제 구현에서 제외 가능성 있음.

## 3. 유물 시스템 (03.RelicData)
유물은 게임 플레이 중 지속적인 패시브 효과나 특정 조건 만족 시 보너스를 제공합니다.

### 3.1 유물 데이터 활용 기준
*   **Value_01 ~ Value_10:** 각 유물의 고유 수치. 유물 설명의 `x`값에 순서대로 매칭됨.
    *   예: `relic101` (정밀 타격 오일) - Value_01(3.0)은 파괴된 타일당 추가 점수(+3점)를 의미함.
*   **단위 구분:** 수치에 따라 정수(점수, 골드), 실수(배율), 만분율(확률, 100% = 10000)로 구분하여 적용.

### 3.2 주요 유물 카테고리 (계열)
*   **도구 계열 (100번대):** 도구 태그 카드 점수 강화, 파괴 방지, 내구도 관련.
*   **잔상/균열 계열:** 특정 타일(잔상, 균열) 파괴 시 시너지 효과.
*   **마법 계열 (200번대):** 마나 획득, 마법 카드 마나 감소, 속성(화염/얼음/전기) 부여.

## 4. 카드 태그 및 비트 연산 (Tag_bit)
에이전트는 카드의 성능을 판별할 때 `Tag_bit`의 합산 값을 분석해야 합니다.

*   **계산 방식:** 각 태그의 비트 값을 더함 (Bitwise OR).
    *   예: **황금 삽 (Tag_bit 96)** = 도구(32) + 황금(64)
    *   예: **균열 카드 (Tag_bit 128)** = 일회성(128)
*   **태그 목록:**
    1.  도구 (32): 유물 `relic101~105` 등과 상호작용.
    2.  황금 (64): 점수 기대값이 높거나 골드 관련 시너지.
    3.  일회성 (128): 사용 후 덱에서 소멸하는 카드.
    4.  마법 (512): 마나를 소모하며 유물 `relic205~212` 등의 영향 하에 있음.

## 5. 시스템 간 연관 관계
1.  **카드 ↔ 유물:** 유물은 특정 `Tag_bit`를 가진 카드의 점수(Value), 파괴 확률, 마나 소모량을 실시간으로 수정함.
2.  **카드 ↔ 강화:** `Card_Upgraded_Type`을 통해 상점이나 이벤트에서 카드가 교체될 대상(`Card_Upgraded_Key`)을 결정함.
3.  **캐릭터 ↔ 카드/유물:** `Char_bit`에 따라 해당 캐릭터의 보상 풀(Pool)에 포함될지 결정됨.
