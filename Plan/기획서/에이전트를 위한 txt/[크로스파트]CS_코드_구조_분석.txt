■ Managers 폴더 — 핵심 시스템 관리자

- ExcelManager.cs: 데이터 총괄 관리자. 게임 시작 시 모든 테이블 로드 및 캐싱. 가장 중요한 파일
- DeckManager.cs: 덱/핸드 관리자. 핸드 배분, 카드 사용/소각, 묘지 관리
- PlayerManager.cs: 플레이어 상태 관리자. 유물 이벤트 시스템(TurnEventSystem), 점수 배율 등 런타임 스탯 관리
- TileGameManager.cs: 타일 게임 관리자. 타일 클릭/발굴, 스테이지 종료 판정
- GameManager.cs: 씬 전환, 게임 전체 흐름 제어
- AccountManager.cs: 로그인/계정 관련
- SeedManager.cs: 랜덤 시드 관리 (재현 가능한 랜덤용)
- ResourceLoadManager.cs: 리소스(이미지, 사운드 등) 로드
- UIManager.cs: UI 전반 제어
- SoundManager.cs: 사운드 재생

■ Data 폴더 — 데이터 테이블 정의

모든 데이터는 ScriptableObject 형태로 유니티에 저장되며 엑셀에서 임포트됨

- ToolExcelDataTable.cs: 카드(도구) 기본 스펙 (카드 키, 이름/설명, 타입, 희귀도, 태그(비트), 마나 코스트, 추가옵션 ID, 이미지 ID)
- ToolTable.cs: 카드의 그리드 형태/배치 (도구 모양)
- ToolCardOptionTable.cs: 카드 추가 옵션(부가효과) (옵션 ID, 효과값)
- ToolTag.cs: 카드 태그 비트 열거형 (소각 128비트, 균열 32비트 등)
- TagTable.cs: 태그 설명 데이터 (태그 ID, 이름/설명)
- GachaTable.cs: 가챠 확률표 (가챠 그룹 ID, 확률(rate), 보상 타입/값)
- PackTable.cs: 카드팩 정보 (팩 ID, 연결 가챠 키, 카드 표시 수, 선택 수)
- ShopTable.cs: 상점 슬롯 구성 (맵 키, 슬롯 인덱스, 연결 가챠 ID)
- ShopItemTable.cs: 상점 개별 아이템 (아이템 타입, 아이템 ID, 최소/최대 가격)
- MapTable.cs: 맵/던전 정보 (맵 이름, 상점 리롤 비용, 스테이지 보상 가챠 ID)
- RequestTable.cs: 의뢰(퀘스트) 정보 (등급, 초기비용, 최대투자, 성공률, 방문당 성공률 증가, 투자당 성공률, 대박률, 실패/성공 보상박스 ID)
- BoxTable.cs: 보상 박스 정의 (박스 ID, 선택 개수)
- BoxItemTable.cs: 박스 안 아이템 목록 (박스 ID 기준으로 아이템 나열)
- LocalizeTable.cs: 다국어 텍스트 (키 → 텍스트 매핑)
- RelicData/RelicDataTable.cs: 유물 기본 스펙 (유물 이름(열거형), 다국어 키, 희귀도, 수치값 배열(Values[]))

■ 핵심 시스템 1 — 데이터 테이블 로드 구조

게임 시작
    → ExcelManager (Awake)
        → Addressables로 14개 테이블 순차 비동기 로드
            (ToolTable, RelicDataTable, ToolExcelDataTable,
             ToolCardOptionTable, GachaTable, TagTable,
             LocalizeTable, PackTable, ShopItemTable,
             ShopTable, MapTable, RequestTable, BoxTable, BoxItemTable)
        → 각 테이블을 Dictionary로 캐싱 (ID → 데이터 객체)
        → initialized = true → 다른 시스템이 데이터 조회 시작 가능

핵심 포인트:
- 모든 게임 데이터는 유니티 에디터 내 ScriptableObject 에셋에 저장됨
- 게임 실행 시 Addressables 시스템으로 비동기 로드 (빌드 크기 최적화)
- 로드 완료 후 Dictionary에 캐싱하여 이후 조회는 O(1) 속도
- 가챠 실행(ExecuteGacha)도 ExcelManager가 담당 — 난수로 가챠 그룹 내 결과 결정

기획자 참고: 엑셀에서 수정한 데이터를 게임에 반영하려면 임포터로 ScriptableObject를 업데이트해야 함

■ 핵심 시스템 2 — 카드 시스템

카드의 흐름:
덱(trueDeck) → 핸드(hand, 최대 handMax장) → 사용 → 묘지(graveyard)
                                            ↓
                                        태그에 따라:
                                        - 소각(128비트): 덱에서 완전 제거
                                        - 균열(32비트): 사용횟수에 따라 확률적 파괴
                                        - 일반: 묘지로만 이동

관련 파일:
- ToolExcelDataTable.cs: 카드 이름/마나/태그/희귀도 등 원본 데이터
- DeckManager.cs 내 CardInGame: 실제 게임 중 카드 인스턴스 (사용횟수, 덱 ID 보유)
- DeckManager.cs: 초기 셔플, 핸드 구성, 카드 선택, 사용 처리
- CardOptionExecuter.cs: 옵션 ID에 따라 실행기 선택 (팩토리 패턴)
- RandomExecuter.cs: 무작위 효과 실행
- CrackExecuter.cs: 균열 관련 효과
- DarkLitExecuter.cs: 어두운 타일 관련 효과
- CardPart.cs / CardPopupParts.cs: 카드 UI 렌더링

기획자 참고:
- 카드 태그는 비트 플래그 방식 (여러 태그 동시 보유 가능)
- 소각 태그(128비트)가 붙으면 사용 즉시 덱에서 영구 제거
- 균열 태그(32비트)는 사용할수록 파괴 확률이 올라가는 방식 (최대 14회 기준)
- 핸드 최대 장수는 기본값 + ExpandedBackpack 유물 보유 시 +1

■ 핵심 시스템 3 — 유물 시스템

유물 획득 흐름:
유물 획득
    → RelicCommandData.MakeRelicCommandData(data)
        → RelicName 기준으로 팩토리에서 해당 유물 클래스 생성
        → InitCommandData(data) → rawData.Values[] 수치 주입
        → PlayerManager.TurnEventSystem에 이벤트 등록
            → 게임 중 해당 이벤트 발생 시 효과 자동 발동

■ 핵심 시스템 4 — 상점/의뢰 시스템

[상점 구조]
MapData (맵별 상점 설정)
    → shopRerollCost: 리롤 비용
    → stageCardRewardGachaId: 스테이지 보상 가챠

ShopData (상점 슬롯 배치)
    → mapKey: 어느 맵의 상점인지
    → slotIndex: 슬롯 번호
    → gachaId: 이 슬롯에서 어떤 가챠로 아이템 결정

ShopItemData (상점 아이템 개별 스펙)
    → itemType: 아이템 타입 (카드팩 or 유물)
    → itemId: 아이템 식별자
    → costMin/costMax: 가격 범위 (랜덤 결정)

PackData (카드팩 스펙)
    → gachaKey: 이 팩의 카드 가챠 테이블
    → cardShow: 보여주는 카드 수
    → cardPick: 선택 가능한 카드 수

[의뢰 시스템 RequestTable 필드 정의]
- requestGrade: 의뢰 등급
- cost: 의뢰 참여 초기 비용
- maxInvest: 최대 투자 한도
- successRate: 기본 성공률
- successRatePerVisit: 상점 방문 횟수당 성공률 증가량
- successRateInvest: 투자 1회당 성공률 증가량
- jackpotRate: 대박 확률
- jackpotRateInvest: 투자당 대박률 증가량
- failBoxId: 실패 시 보상 박스 ID
- successBoxId: 성공 시 보상 박스 ID

[보상 박스 구조]
BoxData (박스 정의)
    → boxId: 박스 식별자
    → pickCount: 몇 개 선택 가능한지 (0 = 전부 지급)

BoxItemData (박스 안 아이템 목록)
    → boxId: 어느 박스 소속인지
    → 아이템 정보들

기획자 참고:
- 상점 가격은 costMin~costMax 사이에서 매번 랜덤 결정됨
- 의뢰 성공률은 기본값 + 방문횟수 보너스 + 투자 보너스의 합산
- 보상은 박스 → 아이템 목록으로 2단계 구조 (박스 ID로 가챠풀 분리 가능)
- 상점 UI 메인: UIStoreView.cs (가장 큰 UI 파일)

■ 가챠 시스템 (공통)

가챠는 상점, 카드팩, 스테이지 보상, 의뢰 보상 모두에서 공통으로 사용됨

GachaTable
    → ExcelManager.ExecuteGacha(gachaId)
        → gachaId 그룹의 누적확률 배열에서 난수로 결과 결정
        → GachaData 반환 (rewardType, rewardValue, rewardCount)

Reward_Type 인덱스:
- 1: 골드
- 2: 카드 (02.CardData 참조)
- 3: 유물 (03.RelicData 참조)
- 4: 팩 (06.PackData 참조)
- 5: 예약 (오입력 방지용 공백)
- 6: 가챠 (07.GachaData 참조)
